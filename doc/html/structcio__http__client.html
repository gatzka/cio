Detailed Description

A cio_http_client struct represents an HTTP client connection.

A cio_http_client is automatically created by the cio_http_server after a successfully established TCP connection. The memory allocation is done by the alloc_client function provided during initialization of a cio_http_server. The resources allocated for the HTTP client are automatically freed when the client connection is closed by using the free_client function provided during initialization of a cio_http_server.

void(* cio_http_client::add_response_header) (struct cio_http_client *client, struct cio_write_buffer *wbh)

Queues an additional HTTP header entry.

This function queues an additional HTTP header entry which will be sent when an HTTP response is sent. Calling this function does not cause any data sent to the client!

Warning
The user of this function must take care that a complete header line is given in wbh! Otherwise the whole header might not be valid HTTP!
The data wbh points to must be available and valid until the complete HTTP reponse was sent or the connection to the client is closed. The response was sent when the response_written callback function of write_response was called.
Parameters
clientThe client which shall get the HTTP header response entry.
wbhThe write buffer head pointing to the data of the header line.

enum cio_error(* cio_http_client::write_response) (struct cio_http_client *client, enum cio_http_status_code status_code, struct cio_write_buffer *wbh_body, cio_response_written_cb_t written_cb)

Writes a non-chunked response to the requesting client.

This function writes a non-chunked response to the requesting client. This function takes care itself to add Content-Length and Connection header fields. Additional header fields can be added by calling add_response_header.

Warning
Any data belonging to response (namely the body data wbh_body points to and all header lines added by add_response_header) must be available and valid until the complete HTTP response was sent or the connection to the client is closed. The response was sent when the response_written callback function was called.
Parameters
clientThe client which shall get the response.
status_codeThe http status code of the response.
wbh_bodyThe write buffer head containing the data which should be written after the HTTP response header to the client (the http body).
response_writtenAn optional callback that will be called whe the response was written. If err != CIO_SUCCESS, the client will be automatically closed.
Returns
CIO_SUCCESS for success. If return value not CIO_SUCCESS, this typically means that you tried to send two responses per request.
